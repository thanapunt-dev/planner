# วิธีติดตั้ง — แผนผลิตนาวังรายสัปดาห์ (เวอร์ชันไม่ใช้ Google Sheet)

เวอร์ชันนี้เก็บทุกอย่างไว้บน Vercel เอง (ใช้ฟีเจอร์ **Vercel Blob** เก็บรูป+ข้อมูล) ไม่ต้องมี Google Sheet/Drive/Apps Script อีกต่อไป
ทุกคนที่เปิดลิงก์เว็บแอปจะเห็นรูปชุดเดียวกันทันที และกดปุ่ม "+ เพิ่มรูป" อัปโหลดได้จากมือถือ ใช้งานได้เลย ไม่ต้องตั้งค่าอะไรเพิ่ม

## ไฟล์ในชุดนี้

```
index.html          ← หน้าเว็บแอป
package.json        ← ระบุ dependency (@vercel/blob)
api/list.js          ← ดึงรายการรูปทั้งหมด
api/upload.js        ← รับรูปที่อัปโหลด บันทึกลง Blob
api/delete.js        ← ลบรูป
```

## ขั้นตอนที่ 1: อัปโหลดขึ้น GitHub

สร้าง repo ใหม่ (หรือใช้ repo เดิมที่เคยทำ) แล้วอัปโหลดไฟล์ทั้งหมดในชุดนี้ขึ้นไป — ต้องคงโครงสร้างโฟลเดอร์ตามเดิม (ไฟล์ `list.js`, `upload.js`, `delete.js` ต้องอยู่ในโฟลเดอร์ชื่อ `api` เป๊ะๆ)

## ขั้นตอนที่ 2: Import เข้า Vercel

1. เข้า https://vercel.com → **Add New... > Project**
2. เลือก repo ที่เพิ่งอัปโหลด → กด **Deploy** (ไม่ต้องตั้งค่าอะไรเพิ่ม Vercel จะรู้จักโฟลเดอร์ `api` เองอัตโนมัติ)

## ขั้นตอนที่ 3: เปิดใช้ Vercel Blob (ทำครั้งเดียว)

1. เข้าโปรเจกต์ที่เพิ่ง deploy ใน Vercel Dashboard
2. ไปแท็บ **Storage** → **Create Database** → เลือก **Blob**
3. ตั้งชื่อ store (เช่น `nawang-plan-photos`) แล้วกด **Create** → กด **Connect** เพื่อเชื่อมกับโปรเจกต์นี้ (Vercel จะเพิ่มตัวแปรที่จำเป็นให้อัตโนมัติ ไม่ต้องคัดลอกคีย์เอง)
4. กลับไปแท็บ **Deployments** → กด **Redeploy** ที่ deployment ล่าสุด เพื่อให้โปรเจกต์รับค่าตัวแปรใหม่ (ต้องทำทุกครั้งหลังเชื่อม/เปลี่ยน Blob store)

เท่านี้เสร็จ — เปิดลิงก์เว็บแอปที่ Vercel ให้มา ก็ใช้งานได้ทันที ไม่ต้องตั้งค่าอะไรในหน้าเว็บอีก

## หมายเหตุ

- รูปที่อัปโหลดจะถูกย่อขนาด (กว้างสูงสุด ~1800px) ก่อนส่งขึ้น เพื่อให้อัปโหลดเร็วและยังคมชัดพออ่านตัวเลขในตารางได้ (ไฟล์ที่ย่อแล้วมักเล็กกว่าขีดจำกัดของ Vercel มาก จึงไม่ต้องกังวลเรื่องขนาดไฟล์)
- ข้อมูล "รายการรูปทั้งหมด" เก็บเป็นไฟล์ `manifest.json` หนึ่งไฟล์ใน Blob store เดียวกัน — ระบบจะสร้างให้เองตอนอัปโหลดรูปแรก ไม่ต้องสร้างเอง
- Vercel Blob ใช้ฟรีได้ในแผน Hobby (มีโควตาการใช้งานต่อเดือน) เพียงพอกับปริมาณรูปแผนผลิตรายสัปดาห์ทั่วไป
- ถ้าอัปโหลดแล้วขึ้น error เกี่ยวกับ token หรือ Blob store ให้เช็คว่าทำขั้นตอนที่ 3 (เชื่อม Blob store + Redeploy) ครบหรือยัง
